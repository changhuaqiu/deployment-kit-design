# 页面设计说明（Desktop-first）

## 全局设计（适用于所有页面）
- Layout：桌面优先，主体采用 12 栏栅格（CSS Grid），左右留白；局部模块用 Flex 排列。
- 响应式：>=1200px 三栏/双栏；768–1199px 变为单栏 + 抽屉；<768px（可选）仅保证可读与可点击。
- Global Styles（像素游戏化 Token）：
  - 背景：#0B1020（深夜蓝）+ 像素噪点纹理；卡片：#121A33；描边：#2B3563（2px 硬边）。
  - 字体：标题用像素风展示字体（fallback 到系统字体），正文用无衬线；字号阶梯：12/14/16/20/24。
  - 主色：#6CFFB3（荧光绿）；警示：#FF5C7A；信息：#5CC8FF。
  - 按钮：像素描边+阴影位移（hover 位移 1px）；禁用态灰化并显示锁图标。
- 交互语言：把“变更”表达为“任务卡/关卡”，把“执行”表达为“出征/战斗”，但在关键位置保留专业术语（Diff/Apply/回滚）。

---

## Page 1：变更地图（首页）（/map）
- Meta：Title「变更地图」；OG 图为像素地图缩略图。
- Layout：左侧“地图画布区”（约 65%），右侧“任务面板”（约 35%）。
- Page Structure：顶部导航 + 主体双栏。
- Sections & Components：
  1. 顶部导航栏：环境切换（dev/stage/prod）、搜索框、重置演示（原型）。
  2. 地图画布区：
     - 像素地图（静态/轻动效），环境作为“区域”；变更作为“关卡节点（可点击）”。
     - 节点状态：草稿灰、待审黄、已批绿、执行中蓝闪、失败红。
     - 悬浮信息卡：标题、影响资源数、创建者、更新时间。
  3. 任务面板：
     - 变更列表（表格/卡片切换）：按状态筛选、按更新时间排序。
     - “创建任务”按钮：弹窗（Modal）。
  4. 创建任务弹窗：选择任务类型（现网→生成 IaC / 现网+IaC→同步 / 从 0→生成 IaC）、标题、目标环境、范围/仓库占位；提交后生成任务卡并进入 IaC 工坊。

## Page 2：IaC 工坊（导入/生成/同步）（/workshop/:id）
- Meta：Title「IaC 工坊」；Description「从现网生成/同步 IaC」。
- Layout：三栏：左侧“任务步骤/背包”、中部“工作台（文件/资源）”、右侧“风险/成本/策略”。
- Sections & Components：
  1. 步骤条：扫描现网 → 归属标注 → 生成/同步 → 预览补丁 → 打包输出。
  2. 资源清单（Inventory）：树形/表格，支持筛选与标记（托管/忽略/只读），展示依赖关系。
  3. 输出区：
     - 生成 IaC：显示目录树与关键文件预览；支持下载或写入仓库（后续）。
     - 同步 IaC：显示 patch（文件 diff）与 drift 分类，支持“仅修复 IaC / 仅标记漂移 / 生成修复分支”。
  4. 右侧面板：策略检查结果、风险标签、成本变化、注意事项（如不可表达字段）。

## Page 3：变更关卡（详情/评审）（/changes/:id）
- Meta：Title「变更关卡：{title}」；Description 包含目标环境与风险等级。
- Layout：三段式：顶部摘要条 + 中部双栏（Diff / 评审）+ 底部操作条。
- Sections & Components：
  1. 顶部摘要条（Sticky）：状态徽章、影响资源数、风险提示（红色像素旗帜）、关卡进度条（草稿→待审→已批→执行中→完成）。
  2. Diff 面板（左）：
     - Tabs：新增/修改/删除；每项为可展开“资源卡”。
     - 高风险标记：删除/权限扩大项默认置顶并加警示边框。
     - “展开全部/收起全部”。
  3. 评审面板（右）：
     - 评论流（Reviewer/Submitter 区分气泡样式）。
     - 决策区：通过/拒绝按钮；拒绝需填写原因；通过后显示“已解锁执行”。
  4. 底部操作条：
     - 主按钮“开始执行（Apply）”仅在已批状态可用；点击后进入战报页。
     - 次按钮：返回地图、复制链接。
  5. 原型交互要求：状态流转必须可见（按钮启用/禁用、节点颜色变化、进度条推进）。

## Page 4：部署战报（运行/历史）（/runs/:id）
- Meta：Title「部署战报」；OG 为像素“胜利/失败”徽章。
- Layout：左侧时间轴 + 右侧日志/摘要卡；顶部展示结果横幅。
- Sections & Components：
  1. 结果横幅：成功（绿）/失败（红）；显示耗时、执行人、目标环境。
  2. 时间轴：步骤卡（语法校验/计划生成/测试部署/生产试运行/验证/完成）；当前步骤高亮并带像素指示箭头；支持模拟推进。
  3. 日志与错误定位：关键日志片段（可折叠）；失败时展示“错误原因卡 + 建议动作”。
  4. 回滚区：主按钮“回滚”；二次确认弹窗（说明影响）；原型为模拟结果并生成复盘记录。
  5. 复盘区：文本框 + 标签（原因：配置错误/权限/依赖/未知）；保存后回到地图并在该变更上显示“已复盘”小图标。
