# IaC城市规划可视化系统 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**CityBuilder** - Infrastructure as Code 城市规划可视化系统

### 1.2 产品定位
将云基础设施的 IaC (Infrastructure as Code) 部署过程，转化为直观的**3D城市建造游戏**体验，让开发者和运维人员能够可视化地理解、管理和部署基础设施。

### 1.3 目标用户

| 用户类型 | 角色 | 核心痛点 | 使用场景 |
|---------|------|----------|----------|
| **云平台开发者** | 应用开发 | 不理解基础设施架构 | 代码审查时查看影响范围 |
| **DevOps工程师** | 运维部署 | 部署失败难以定位问题 | 实时监控部署进度 |
| **技术负责人** | 架构决策 | 难以评估变更影响 | 审查架构变更方案 |
| **新手工程师** | 学习者 | IaC学习曲线陡峭 | 可视化学习基础设施概念 |

### 1.4 核心价值主张

> **"让基础设施部署像玩模拟城市一样直观"**

1. **降低认知门槛** - 用城市隐喻解释技术概念
2. **提升可视化能力** - 实时看到部署过程和资源状态
3. **减少配置错误** - 违建稽查功能发现配置漂移
4. **加速决策过程** - 直观的影响评估和成本展示

## 2. 核心功能

### 2.1 三大核心模式

#### 模式1: 城市普查 (Census Mode)
**对应场景**: 现网资源 → IaC 代码（反向生成）

**用户流程**:
```
1. 用户点击"城市普查"按钮
2. 系统扫描现网资源（调用MCP）
3. 地图上逐渐浮现建筑（动画效果）
4. 生成"城市蓝图 v1"（IaC代码）
5. 用户可以审查和编辑蓝图
```

**价值**: 快速将现有基础设施转化为代码，实现 IaC 转型

#### 模式2: 违建稽查 (Violation Inspection)
**对应场景**: 配置漂移检测

**用户流程**:
```
1. 用户点击"违建稽查"按钮
2. 系统对比现网 vs IaC 代码
3. 漂移建筑高亮显示（黄色 + ⚠️）
4. 点击建筑查看差异详情
5. 用户选择：同步到蓝图 or 回滚现网
```

**价值**: 发现并修复配置漂移，保持代码与现网一致

#### 模式3: 新区规划 (District Planning)
**对应场景**: 从零创建基础设施

**用户流程**:
```
1. 用户选择建筑工具（右侧面板）
2. 地面显示半透明预览
3. 点击放置建筑（弹跳动画）
4. 系统实时估算成本
5. 点击"开始施工"执行部署
```

**价值**: 直观地规划和创建新基础设施

### 2.2 施工许可链路 (Deployment Workflow)

将技术部署流程转化为城市建设的5个阶段：

| 阶段 | 城市隐喻 | 技术对应 | 视觉表现 |
|------|----------|----------|----------|
| **图纸审查** | 检查建筑图纸 | IaC语法校验 | ✅ 绿色对勾 |
| **影响评估** | 评估施工影响 | 生成执行计划 | ✅ 绿色对勾 |
| **沙盘推演** | 测试环境模拟 | 测试环境部署 | 🔄 进度动画 |
| **试通车** | 灰度发布 | 生产环境试运行 | ⏳ 等待图标 |
| **竣工验收** | 最终验收 | 确认部署成功 | ⏳ 等待图标 |

### 2.3 辅助功能

#### 建筑详情面板
- **建筑部件**: 显示内部资源（CPU、内存、存储）
- **建筑配套**: 显示 Traits（高可用、自动扩容）
- **配置差异**: 并排对比现网 vs 蓝图
- **IaC 代码**: 显示生成的 Terraform/CloudFormation 代码

#### 城市统计
- 建筑数量
- 电力使用率（对应资源利用率）
- 维护费用（对应云服务费用）
- 可用性百分比

#### 多环境支持
- 开发环境（蓝色天空）
- 测试环境（橙色天空）
- 生产环境（灰色天空 + 更多建筑）

## 5. 非功能需求

### 5.1 性能要求
- **首屏加载**: < 3秒
- **3D 渲染**: 稳定 60 FPS
- **建筑数量**: 支持 500+ 建筑
- **WebSocket 延迟**: < 100ms
- **内存占用**: < 500MB

### 5.2 兼容性要求
- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **WebGL**: 需要 WebGL 1.0+ 支持
- **分辨率**: 1920x1080 最佳，支持 1366x768 最低
- **移动端**: iOS 14+, Android 10+ (P3)

### 5.3 安全要求
- **数据加密**: WebSocket TLS 1.3
- **访问控制**: 集成企业 SSO
- **审计日志**: 记录所有操作
- **权限隔离**: 不同环境权限分离

### 5.4 可用性要求
- **无障碍**: WCAG 2.1 AA 级
- **国际化**: 支持中英文
- **错误处理**: 友好的错误提示
- **帮助文档**: 内置交互式教程

## 7. 风险与限制

### 7.1 技术风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| WebGL 不兼容 | 高 | 提供 2D 降级方案 |
| 性能瓶颈 | 中 | 使用实例化渲染、LOD |
| 浏览器差异 | 中 | 充分测试主流浏览器 |

### 7.2 产品风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 隐喻不准确 | 高 | 用户测试验证隐喻 |
| 学习成本 | 中 | 提供交互式教程 |
| 用户接受度 | 低 | 渐进式功能开放 |

### 7.3 业务风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 开发周期长 | 中 | 分阶段交付 |
| 维护成本高 | 中 | 模块化架构 |
| 推广难度 | 低 | 内部试点先行 |
