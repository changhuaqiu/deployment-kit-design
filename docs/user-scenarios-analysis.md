## 详细场景分析

### 场景A: 现网资源 → IaC 代码（反向生成）

**用户目标**: 将现有基础设施转化为代码，实现 IaC 转型

**前置条件**:
- 现网有运行中的资源
- 没有 IaC 代码或代码过时

**操作流程**:
```
1. 用户选择应用/项目
2. 系统扫描现网资源 (MCP 调用)
   ├─ ADS 集群
   ├─ 配置
   ├─ VPC
   ├─ 安全组
   └─ 负载均衡等
3. 用户预览扫描结果
4. 生成 IaC 代码
5. 用户审查生成的代码
6. 保存到代码仓库
```

**可视化需求**:
- 🗺️ **资源地图**: 显示现网资源分布（类似战略地图）
- 📊 **资源清单**: 列表视图，按类型分组
- 🔄 **生成进度**: 实时显示代码生成进度
- ✅ **验证状态**: 显示生成的代码是否符合规范

**关键决策点**:
- 哪些资源需要导入？
- 生成的代码是否需要手动调整？
- 是否立即提交到代码仓库？

---

### 场景B: 检测配置漂移

**用户目标**: 发现并修复现网与代码的不一致

**前置条件**:
- 现网有资源
- 有对应的 IaC 代码
- 可能存在配置漂移

**操作流程**:
```
1. 用户选择应用/项目
2. 系统同时获取:
   ├─ 现网资源状态 (MCP 实时调用)
   └─ IaC 代码定义 (本地文件)
3. 执行差异分析 (Diff Algorithm)
4. 可视化展示差异:
   ├─ 🟢 新增资源 (现网有，代码无)
   ├─ 🟡 配置变更 (现网 vs 代码不一致)
   └─ 🔴 缺失资源 (代码有，现网无)
5. 用户选择处理方式:
   ├─ 自动更新 IaC 代码
   ├─ 手动确认每个变更
   └─ 忽略某些差异
6. 应用变更
```

**可视化需求**:
- 🔍 **对比视图**: 并排显示现网 vs 代码
- 🎨 **颜色编码**: 绿色(新增)、黄色(变更)、红色(删除)
- 📝 **变更详情**: 点击查看具体字段变更
- ✅ **应用预览**: 显示应用后的预期状态

**关键决策点**:
- 哪些差异需要同步？
- 同步方向：代码→现网 还是 现网→代码？
- 是否需要人工审核每个变更？

---

### 场景C: 从零创建资源

**用户目标**: 创建新的基础设施并生成代码

**前置条件**:
- 新项目或扩展现有项目
- 需要定义资源规格

**操作流程**:
```
1. 用户启动创建向导
2. 选择资源类型:
   ├─ ADS 集群
   ├─ 数据库
   ├─ 缓存
   └─ 其他
3. 配置资源参数:
   ├─ 规格 (flavor)
   ├─ 数量
   ├─ 网络配置
   └─ 其他参数
4. 实时预览成本和影响
5. 生成 IaC 代码
6. （可选）直接部署
```

**可视化需求**:
- 🎛️ **配置面板**: 表单化输入参数
- 💰 **成本估算**: 实时显示预计成本
- 🗺️ **架构预览**: 显示新资源在整体架构中的位置
- 📋 **代码预览**: 实时生成的代码

**关键决策点**:
- 资源规格是否合理？
- 是否符合组织规范？
- 是否需要审批？

---

### 场景D: IaC 语法校验

**用户目标**: 确保代码语法正确，符合规范

**前置条件**:
- 已有 IaC 代码
- 准备部署

**操作流程**:
```
1. 用户提交 IaC 代码
2. 系统执行语法校验:
   ├─ 语法检查
   ├─ 命名规范
   ├─ 安全规范
   └─ 最佳实践
3. 显示校验结果:
   ├─ ✅ 通过
   ├─ ⚠️ 警告
   └─ ❌ 错误
4. 用户修复问题
5. 重新校验
```

**可视化需求**:
- 📊 **校验面板**: 显示所有检查项
- 🎯 **问题定位**: 点击跳转到错误位置
- 📈 **合规分数**: 显示代码质量评分
- 🔄 **实时反馈**: 编辑时即时校验

**关键决策点**:
- 警告是否阻止部署？
- 是否需要自动修复？

---

### 场景E: 创建执行计划

**用户目标**: 了解部署将产生的影响

**前置条件**:
- IaC 代码已校验通过
- 准备部署

**操作流程**:
```
1. 用户选择目标环境
2. 系统分析变更:
   ├─ 读取当前状态 (现网或缓存)
   ├─ 解析目标状态 (IaC 代码)
   └─ 计算差异
3. 生成执行计划:
   ├─ 资源创建顺序
   ├─ 依赖关系图
   ├─ 预计耗时
   └─ 风险评估
4. 用户审查计划
5. （可选）调整计划
6. 确认执行
```

**可视化需求**:
- 📊 **执行计划图**: DAG 图显示依赖关系
- ⏱️ **时间线**: 显示每个步骤的预计时间
- 🎨 **变更高亮**: 清晰标记将要变更的资源
- 📈 **影响评估**: 显示影响范围和风险

**关键决策点**:
- 执行顺序是否合理？
- 是否在维护窗口内？
- 是否需要分批执行？

---

### 场景F: 测试部署

**用户目标**: 在测试环境验证部署

**前置条件**:
- 执行计划已确认
- 测试环境就绪

**操作流程**:
```
1. 用户启动测试部署
2. 系统执行部署流程 (实时更新):
   ├─ 创建/更新资源
   ├─ 健康检查
   ├─ 配置验证
   └─ 集成测试
3. 实时监控部署状态:
   ├─ 🟢 进行中
   ├─ ✅ 成功
   ├─ ⚠️ 警告
   └─ ❌ 失败
4. 失败时回滚
5. 生成部署报告
```

**可视化需求**:
- 🗺️ **部署地图**: 实时显示资源部署进度
- 📊 **进度面板**: 总体进度和当前步骤
- 🎬 **动画效果**: 资源创建/更新动画
- 📋 **日志流**: 实时日志输出
- 🔔 **告警通知**: 失败时立即通知

**关键决策点**:
- 部署失败是否自动回滚？
- 哪些警告可以忽略？
- 是否需要人工干预？

---

### 场景G: 生产试运行

**用户目标**: 在生产环境安全地部署

**前置条件**:
- 测试部署成功
- 获得生产部署批准

**操作流程**:
```
1. 用户选择生产部署模式:
   ├─ 🎯 试运行 (dry-run)
   ├─ 🚀 直接部署
   └─ 🔄 金丝雀发布
2. 执行预检查:
   ├─ 环境检查
   ├─ 依赖检查
   ├─ 容量检查
   └─ 安全检查
3. 执行部署 (分阶段):
   ├─ Stage 1: 10% 流量
   ├─ Stage 2: 50% 流量
   └─ Stage 3: 100% 流量
4. 每阶段监控:
   ├─ 资源健康
   ├─ 性能指标
   ├─ 错误率
   └─ 用户反馈
5. 异常时回滚
6. 完成后生成报告
```

**可视化需求**:
- 📊 **阶段进度**: 清晰显示当前阶段
- 🎨 **流量分配**: 可视化流量切换
- 📈 **实时指标**: 显示性能和健康指标
- 🚨 **异常监控**: 异常时醒目提示
- 🔄 **回滚控制**: 一键回滚按钮

**关键决策点**:
- 每个阶段是否继续？
- 性能是否可接受？
- 是否需要回滚？

## 通用 UX 原则

### 1. 渐进式披露
- 默认显示摘要信息
- 用户点击展开查看详情
- 避免信息过载

### 2. 状态可见性
- 实时显示当前状态
- 清晰的进度指示
- 预计剩余时间

### 3. 错误预防
- 关键操作前确认
- 危险操作警告
- 提供撤销选项

### 4. 一致性
- 统一的颜色编码
- 统一的图标系统
- 统一的交互模式

### 5. 快速反馈
- 操作立即响应
- 长时间操作显示进度
- 异步操作状态更新
