# Agent Integration Design: 极简“建筑公司” HUD

## 1. 目标
将 `gastown` (Gas Town) 的多 Agent 协作、工作状态持久化 (Beads) 和合并队列 (Refinery) 概念，无缝融合进当前的 2D “城市建造” HUD 中。
核心诉求：**在 HUD 右侧增加一个极简的 2D 办公室/车间，展示各个施工员 (Worker Agents) 的实时工作状态和质检排队情况，但不做过度复杂的 3D 或大楼切面。**

## 2. 概念映射 (gastown -> 建筑公司)

| gastown 概念 | 极简建筑公司隐喻 | 职责与呈现 |
| :--- | :--- | :--- |
| **Mayor (你)** | **包工头 / 市长** | HUD 的真实操作者。负责点击“普查/代码/施工”按钮，向车间下发宏观指令。 |
| **Polecats** | **施工员 (如 🕵️普查员, 👨‍🎨绘图员)** | 在右侧车间工位上的常驻 Avatar。负责具体的代码生成、资源扫描等任务。 |
| **Deacon/Witness** | **👮 监工 / 安检员** | 车间角落的质检员。负责健康检查、拦截危险漂移 (Drift)。 |
| **Refinery / Convoys** | **质检台 / 工程单堆** | 车间内的一张桌子或短传送带。施工员做完的“图纸 (Beads)”会被放在这里，等待监工审核或包工头签字。 |
| **Hooks / Beads Ledger** | **工程台账抽屉** | 点击工人或图纸后，从左侧弹出的详细日志面板（显示具体的思考链和执行证据）。 |

## 3. UX 布局与组件设计

### 3.1 右侧面板划分 (`CityHudMap.tsx`)
右侧 HUD 面板（目前是 108px 宽的工具箱）将垂直拆分为上下两部分：
*   **上半部分 (30%) - 🏗️ 规划工具 (Tools):** 保留现有的建筑放置工具（办公楼、金库、道路等）。
*   **下半部分 (70%) - 👷 建筑公司 (Agent Office):** 新增的多 Agent 监控区。

### 3.2 建筑公司 (Agent Office) 内部结构
该区域采用极简的 2D 像素风排版：
1.  **Header:** `👷 建筑公司 (Town Workspace)`
2.  **工位区 (Polecats Zone):**
    *   垂直排列 2-3 个工人的卡片（例如 `🕵️ 普查员`, `👨‍🎨 绘图员`）。
    *   每个卡片包含：
        *   **Avatar**: 像素风小人头。
        *   **Status**: 状态徽章（`Zzz` 空闲, `🔨` 施工中, `🚨` 阻塞）。
        *   **Snippet**: 极短的单行滚动文本，显示当前正在干嘛（例如 `> Scanning AWS...`）。
3.  **质检台 (Refinery Zone):**
    *   面板底部区域。
    *   左侧站着 `👮 监工`。
    *   右侧是 `📥 待审批 (0)` 或 `📥 待合并 (1)` 的队列堆叠图标。
    *   当需要包工头（用户）介入时，这里会闪烁 `🟨 需要签字` 按钮。

### 3.3 工程台账 (Beads Ledger) 抽屉
*   **触发方式:** 点击工位区正在干活的工人，或者质检台上的待审批单。
*   **展现形式:** 左侧弹出一个半透明的抽屉（复用或替换现有的建筑详情抽屉）。
*   **内容结构:**
    *   顶部标题：`📜 工程台账: gt-abc12 (同步 Prod 漂移)`
    *   时间轴列表（类似终端日志）：
        *   `[09:00] 🕵️ 普查员: 发现 3 处违建 (Drift)。`
        *   `[09:01] 👨‍🎨 绘图员: 生成最小修复补丁 (Patch)。`
        *   `[09:02] 👮 监工: 拦截！该补丁将删除关键 DB，要求重新出图！` (Escalation)
        *   `[09:05] 👨‍🎨 绘图员: 提交了修正后的非破坏性补丁。`

## 4. 交互数据流 (模拟多 Agent 协作)

为了在前端原型中生动地展示这个流程，我们需要在 `deployStore.ts` 中引入一个轻量级的 `AgentState` 模拟器。

1.  **用户点击动作栏 (例如 `🔍 稽查`)**
2.  **State 更新:**
    *   `deployStore` 接收指令。
    *   将 `🕵️ 普查员` 的状态设置为 `working`。
    *   触发一个定时器（Tick），每隔 800ms 更新一次工人的 `Snippet` 文本（模拟在跑不同的子任务）。
3.  **任务完成交接 (Handoff to Refinery):**
    *   普查员状态恢复 `idle`。
    *   生成一份 Mock 的 Drift 报告。
    *   将任务丢到 `质检台 (Refinery)`，队列数 +1。
    *   `👮 监工` 状态变为 `thinking`，模拟检查策略。
4.  **包工头审批 (The Mayor's Gate):**
    *   监工检查完毕，质检台亮起 `🟨 需签字`。
    *   用户（包工头）点击确认。
    *   系统正式应用变更，地图上的建筑状态更新。

## 5. 总结与自查
*   [x] 是否抛弃了复杂的大楼切面？是，改为极简的右侧列表/卡片式车间。
*   [x] 是否体现了 `gastown` 的核心精神？是，包含了 Polecats 干活、Refinery 排队、Deacon 监控和 Beads 日志持久化四大要素。
*   [x] 是否与 2D 城市建造 HUD 兼容？是，完美嵌入右侧面板，且与现有的建筑地图逻辑不冲突。
