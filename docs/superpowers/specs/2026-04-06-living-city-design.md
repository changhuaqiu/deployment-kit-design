# 活城市设计：活街区 + 互动办公室系统

**文档版本**: 1.0.0
**创建日期**: 2026-04-06
**作者**: Design Team
**状态**: 设计阶段

---

## 📋 设计概述

### 核心目标

**体验感、代入感、好玩** - 将 IaC 部署过程转化为沉浸式像素城市建造游戏体验

### 设计理念

```
技术准确（Deployment Kit 能力）
    +
游戏化体验（Pixel Agents 互动）
    +
现实约束（AWS/OAM 结构）
    =
活城市系统
```

### 关键设计决策

1. **街区长系统** - 每个街区有角色化主管，主动汇报问题
2. **Agent 流动机制** - Agent 在办公室和街区之间穿梭，创造叙事感
3. **多维度切换** - 资源类型/环境边界/应用边界三个维度
4. **办公室互动** - 点击派工、审批、查看详情
5. **完整游戏循环** - 初始化 → 日常运维 → 成长系统

### 1.2 街区长 + 街区活动系统

#### 街区长角色化设计

**学习 Pixel Agents 的角色化**，每个街区有自己的"主管"：

| 街区类型 | 街区长 | 职责 | 实时状态显示 | 主动事件 |
|---------|--------|------|--------------|---------|
| 🏗️ 计算区 | 👨‍💼 **运营总监** | 监控容器/AI服务健康度 | "🟢 15个实例运行中<br>📊 CPU: 45% \| 内存: 62%" | "🚨 CPU突增！建议扩容" |
| 🏦 数据区 | 👨‍💻 **数据管家** | 管理数据库/数据制品 | "📦 LTS: 85%容量<br>💾 备份: 2小时前" | "📊 LTS容量告警！建议扩容" |
| 🌐 网络区 | 🚦 **交通指挥** | 监控流量/负载均衡 | "🔄 1.2K QPS<br>⚡ 延迟: 23ms" | "🚀 流量突增！考虑增加LB" |
| ⚙️ 配置区 | 📝 **配置管理员** | 检测配置漂移 | "⚠️ 3处配置漂移<br>🔄 最后扫描: 10分钟前" | "⚠️ 发现新漂移！需要修复" |

**关键设计**：
- 街区长**主动汇报**问题（不是被动等待用户查询）
- 气泡提示 + 事件流推送
- 点击街长 → 查看详情/派遣 Agent

### 1.3 Agent 街区巡逻机制（Pixel Agents 风格）

#### Agent 状态机

```typescript
enum AgentState {
  IDLE = 'idle',           // 在办公室闲逛
  WALKING = 'walking',     // 走到目标街区
  WORKING = 'working',     // 在街区执行任务
  RETURNING = 'returning'  // 返回办公室汇报
}

enum AgentRole {
  SCANNER = 'scanner',     // 🕵️ 普查员
  PLANNER = 'planner',     // 👨‍🎨 绘图员
  MONITOR = 'monitor'      // 👮 监工
}
```

#### 完整叙事流程

**场景：发现配置漂移**

```
👮 监工在质检台发现问题
  → 质检台闪烁：📥 待签 (1)
  → 气泡提示："⚠️ 测试城-数据区有漂移"

🗺️ 地图高亮显示
  → 🏦 数据区闪烁黄色 + ⚠️
  → 正常街区保持绿色

🕵️ 普查员接单出发
  → 状态从"Zzz 空闲"变为"🔨 施工中"
  → 从办公室出发（WALK 动画）
  → 路径寻找：办公室 → 测试城-数据区

🕵️ 到达街区开始扫描
  → 到达后状态变为"WORKING"
  → 显示扫描动画（TYPE 动画）
  → 滚动文本："> Scanning LTS DB..."
  → 街区"点亮"（高亮边框）

🕵️ 发现问题！
  → 气泡提示："💬 发现2处配置漂移！"
  → 返回办公室汇报（RETURNING 状态）

👨‍🎨 绘图员生成补丁
  → 在办公室工位上打字（TYPE 动画）
  → 显示"> Generating patch..."

👮 监工审核
  → 检查补丁安全性
  → 如果危险：气泡 "🚨 拦截！将删除关键DB"
  → 如果安全：质检台显示 [🟨 需签字]

✅ 用户签字批准
  → 地图上🏦 数据区执行变更动画
  → 建筑从黄色→绿色
```

### 1.4 视觉特效设计

#### Pixel Agents 风格的动画

| 动画类型 | 描述 | 时长 | 技术实现 |
|---------|------|------|---------|
| **WALK** | Agent 走到目标街区 | 根据距离 | BFS 路径寻找 + 平滑移动 |
| **TYPE** | Agent 执行任务（扫描/生成） | 持续直到完成 | 帧动画（2帧循环） |
| **SPAWN** | Agent 出现（生成） | 1.5s | Matrix 数字雨特效 |
| **DESPAWN** | Agent 消失（完成） | 1.5s | Matrix 反向特效 |
| **BUBBLE** | 气泡提示（发现问题） | 直到点击或超时 | 弹跳动画 + 淡出 |

#### 街区活动特效

- **街区点亮**：Agent 到达时，街区边框高亮（黄色呼吸效果）
- **扫描波纹**：扫描时，从 Agent 位置向外扩散的圆圈
- **建筑生成**：虚线 → 实心 + 弹跳动画（scale: 0 → 1）
- **迷雾散去**：初始地图有雾，扫描后逐渐清晰

### 4.2 路径寻找算法

**使用 BFS（Pixel Agents 同款）**：

```typescript
// 从办公室走到目标街区
function findPathToDistrict(
  start: { col: number; row: number },
  targetDistrict: District,
  tileMap: TileType[][]
): Tile[] {
  // BFS 寻找最短路径
  const targetTiles = getDistrictTiles(targetDistrict);
  const queue: { pos: Tile; path: Tile[] }[] = [{ pos: start, path: [] }];
  const visited = new Set<string>();

  while (queue.length > 0) {
    const { pos, path } = queue.shift()!;

    // 检查是否到达目标街区
    if (isInDistrict(pos, targetDistrict)) {
      return path;
    }

    // 避免重复访问
    const key = `${pos.col},${pos.row}`;
    if (visited.has(key)) continue;
    visited.add(key);

    // 探索相邻格子
    for (const neighbor of getWalkableNeighbors(pos, tileMap)) {
      queue.push({
        pos: neighbor,
        path: [...path, neighbor]
      });
    }
  }

  return []; // 无路径
}
```
